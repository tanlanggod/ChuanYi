# 好物/购物车接口清单（V1补齐）

## 1. 范围与目标
1. 覆盖 Tab `好物` 与 `购物车` 的前后端交互闭环。
2. 与现有 DIY 订单链路并行，不破坏 `定制快照下单`。
3. 提供可直接落库和联调的字段与状态定义。

## 2. 核心对象与字段

### 2.1 好物分类 `goods_category`
| 字段 | 类型 | 必填 | 说明 |

| id | long | 是 | 分类ID |

| name | string(32) | 是 | 分类名 |

## 3. 状态定义

### 3.1 商品状态
1. `ON_SALE`：上架可售。
2. `OFF_SALE`：下架，不可购买。
3. `SOLD_OUT`：SPU维度无可售SKU。

### 3.2 购物车项有效性状态 `validityStatus`
1. `VALID`：有效，可结算。
2. `INVALID_OFF_SALE`：已下架。
3. `INVALID_OUT_OF_STOCK`：库存不足。
4. `INVALID_PRICE_CHANGED`：价格变更需用户确认。
5. `INVALID_NOT_FOUND`：商品或快照不存在。

### 3.3 购物车结算状态 `checkoutStatus`
1. `READY`：可结算。
2. `BLOCKED_EMPTY`：购物车为空。
3. `BLOCKED_NO_SELECTION`：未勾选商品。
4. `BLOCKED_INVALID_ITEMS`：存在失效商品。
5. `BLOCKED_NO_ADDRESS`：未选择地址。
6. `BLOCKED_STOCK_CHANGED`：库存变化需重试。

## 4. 接口清单（好物）

### 4.1 获取好物分类
- 方法：`GET`
- 路径：`/api/goods/categories`
- Query：
  - `withCount` `boolean` 可选，默认 `true`
- 响应 data：
  - `categories: GoodsCategory[]`

### 4.2 获取好物SPU列表
- 方法：`GET`
- 路径：`/api/goods/list`
- Query：
  - `categoryId` `long` 可选
  - `keyword` `string` 可选
  - `pageNo` `int` 必填，默认 `1`
  - `pageSize` `int` 必填，默认 `20`
  - `sortType` `enum` 可选：`DEFAULT`/`PRICE_ASC`/`PRICE_DESC`/`NEWEST`
- 响应 data：
  - `list: GoodsSpu[]`
  - `pageNo/pageSize/total`

### 4.3 获取好物详情（SPU+SKU）
- 方法：`GET`
- 路径：`/api/goods/{spuId}`
- Path：
  - `spuId` `long` 必填
- 响应 data：
  - `spu: GoodsSpu`
  - `skus: GoodsSku[]`

## 5. 接口清单（购物车）

### 5.1 加入购物车
- 方法：`POST`
- 路径：`/api/cart/items`
- 请求 body：
  - `itemType` `enum` 必填：`NORMAL_GOODS`/`DIY_SNAPSHOT`
  - `skuId` `long` 条件必填（普通商品）
  - `snapshotNo` `string` 条件必填（定制商品）
  - `qty` `int` 必填，范围 `1~999`
- 响应 data：
  - `cartItemId` `long`
  - `cartSummary` `CartSummary`

### 5.2 购物车详情
- 方法：`GET`
- 路径：`/api/cart`
- 响应 data：
  - `items: CartItem[]`
  - `summary: CartSummary`
  - `pageState` `enum`：`EMPTY`/`HAS_VALID`/`HAS_INVALID`/`MIXED`

### 5.3 修改购物车数量
- 方法：`PUT`
- 路径：`/api/cart/items/{cartItemId}`
- Path：
  - `cartItemId` `long` 必填
- 请求 body：
  - `qty` `int` 必填，范围 `1~999`
- 响应 data：
  - `cartItem: CartItem`
  - `summary: CartSummary`

### 5.4 勾选/取消单项
- 方法：`PUT`
- 路径：`/api/cart/items/{cartItemId}/select`
- 请求 body：
  - `selected` `boolean` 必填
- 响应 data：
  - `summary: CartSummary`

### 5.5 全选/取消全选
- 方法：`POST`
- 路径：`/api/cart/items/select-all`
- 请求 body：
  - `selected` `boolean` 必填
- 响应 data：
  - `summary: CartSummary`

### 5.6 删除购物车项
- 方法：`DELETE`
- 路径：`/api/cart/items/{cartItemId}`
- 响应 data：
  - `summary: CartSummary`

### 5.7 结算预检查
- 方法：`POST`
- 路径：`/api/cart/settlement/preview`
- 请求 body：
  - `addressId` `long` 可选（前端若已选择地址则传）
  - `cartItemIds` `long[]` 可选（为空则按当前勾选项）
- 响应 data：
  - `summary: CartSummary`
  - `priceLines[]`：`title/amount`
  - `canSubmit` `boolean`
  - `blockingReasons[]` `string`

### 5.8 购物车提交订单
- 方法：`POST`
- 路径：`/api/order/create-from-cart`
- 请求 body：
  - `addressId` `long` 必填
  - `remark` `string(45)` 可选
  - `cartItemIds` `long[]` 可选（为空按当前勾选项）
- 响应 data：
  - `orderNo` `string`
  - `orderStatus` `enum`：`PENDING_PAY`
  - `payAmount` `decimal`
  - `preoccupyStatus` `enum`：`LOCKED`/`FAILED`

## 6. 关键规则（实现侧）
1. 购物车允许同时存在普通商品与定制快照商品，结算时统一走订单主表，订单项按`itemType`区分。
2. 购物车所有价格字段仅展示用途，提交订单时后端按最新SKU/快照重算。
3. 失效项默认不允许进入结算，需先修复或删除。
4. `create-from-cart` 成功后，已下单购物车项从购物车中移除。
5. 定制快照商品数量建议V1限制为1，避免同快照批量下单带来的履约歧义。
